## 🎯 PolyBot とは？

PolyBot は [Polymarket](https://polymarket.com)（世界最大の予測市場プラットフォーム）向けの**完全自動取引ボット**です。

### 主な特徴

| 特徴 | 説明 |
|------|------|
| 🤖 **完全自動化** | 24時間365日取引 |
| 🌐 **Ferox ダッシュボード** | 美しいウェブインターフェースでボットを管理 |
| 🛡️ **リスク管理** | サーキットブレーカー保護と損失制限 |
| 📊 **シグナルエンジン** | 5つのテクニカル指標が連携 |
| 💰 **Solana 資金調達** | 自動 SOL→USDC ブリッジ |
| 🧪 **シミュレーションモード** | 実際のお金を使わずにテスト |

## 🚂 Railway へのデプロイ

### ステップ1：アカウント作成

1. [railway.app](https://railway.app) にアクセス
2. "Login" をクリック
3. "Login with GitHub" を使用（推奨）

### ステップ2：新規プロジェクト作成

1. **"New Project"** をクリック
2. **"Deploy from GitHub repo"** を選択
3. **"PolyBot"** を検索して選択

### ステップ3：環境変数の設定

**Variables** に移動して追加：

| 変数 | 値 | 必須 |
|------|-----|:----:|
| `DRY_RUN` | `true`（テスト用） | 推奨 |
| `MODE` | `signal`、`copy`、`arbitrage` または `all` | オプション |
| `POLYGON_PRIVATE_KEY` | ウォレットの秘密鍵 | 実取引に必須 |

### ステップ4：デプロイの確認

1. デプロイが **緑色** ✅ になるまで待つ
2. Ferox ダッシュボードを開く
3. ボットのステータスを確認
4. オンラインと表示されれば成功！🎉

## 🎯 取引モード

### 1. シグナル取引 (Signal)

シグナルエンジンは5つのテクニカル指標で市場を分析：

- **移動平均線クロスオーバー (MA)** - 27%
- **RSI（相対力指数）** - 27%
- **MACD** - 23%
- **モメンタム** - 13%
- **出来高** - 10%

```bash
# シグナル取引を有効化：
MODE=signal
```

### 2. コピー取引 (Copy)

成功したトレーダーの取引をコピー：

```bash
# コピー取引を有効化：
MODE=copy
COPY_WALLETS=wallet1,wallet2,wallet3
```

### 3. アービトラージ (Arbitrage)

YES + NO < $1.00 の機会を探索：

```bash
# アービトラージを有効化：
MODE=arbitrage
```

### 4. 全モード

```bash
# 全モードを使用：
MODE=all
```

## 🌐 Solana 統合

PolyBot は Solana から自動的に資金を受け取ることができます。

### 設定変数

| 変数 | 説明 | デフォルト値 |
|------|------|-------------|
| `SOLANA_PRIVATE_KEY` | Solana 秘密鍵 | （必須） |
| `AUTO_SWAP_SOL` | 自動 SOL→USDC スワップ | `true` |
| `SOL_SWAP_THRESHOLD` | 最小スワップ SOL | `0.1` |
| `MIN_POLY_BALANCE_USDC` | 自動資金調達の最低残高 | `20.0` |
| `BRIDGE_FUND_AMOUNT` | 最大送金額 | `50.0` |

### 資金調達フロー

1. ボットのアドレスに SOL を送信
2. ボットが SOL を検出
3. Jupiter DEX で SOL → USDC にスワップ
4. Wormhole で USDC を Polygon に転送
5. 取引準備完了！

## 🛡️ リスク管理

PolyBot には複数の保護レイヤーが含まれています：

### サーキットブレーカー

```
┌────────────────────────────────────────────────────────────────┐
│                    🛡️ リスク管理システム                         │
├────────────────────────────────────────────────────────────────┤
│                                                                │
│   ┌──────────────┐    ┌──────────────┐    ┌──────────────┐    │
│   │   日次最大   │    │  サーキット  │    │   ケリー     │    │
│   │    損失      │    │  ブレーカー  │    │  サイジング  │    │
│   │  ───────     │    │  ───────     │    │  ───────     │    │
│   │   -5%        │    │  3回連続     │    │   最適な     │    │
│   │  で停止      │    │  損失で一時  │    │  ポジション  │    │
│   │              │    │  停止        │    │  サイズ      │    │
│   └──────────────┘    └──────────────┘    └──────────────┘    │
│                                                                │
└────────────────────────────────────────────────────────────────┘
```

| 保護機能 | 説明 |
|----------|------|
| **日次損失制限** | -5% の日次損失に達したら取引停止 |
| **サーキットブレーカー** | 3回連続の損失後に自動一時停止 |
| **ケリーサイジング** | エッジに基づいて最適なポジションサイズを計算 |
| **最大ドローダウン** | 設定可能なドローダウン制限 |
| **クールダウン期間** | 保護トリガー後の一時停止期間 |

## 🔧 トラブルシューティング

### よくある問題

| 問題 | 解決策 |
|------|--------|
| ボットが応答しない | Railway ログでエラーを確認 |
| 取引が実行されない | `DRY_RUN=false` とウォレットキーの設定を確認 |
| 残高が 0 と表示される | Polygon ウォレットに USDC があることを確認 |
| API エラー | 環境変数が正しく設定されているか確認 |

### ログの確認

Railway で：
1. プロジェクトに移動
2. "Deployments" をクリック
3. 最新のデプロイを選択
4. "View Logs" をクリック

### 一般的なエラーメッセージ

| エラー | 意味 | 解決策 |
|--------|------|--------|
| `Insufficient balance` | USDC が不足 | ウォレットに資金を追加 |
| `Rate limited` | API リクエストが多すぎる | 数分待つ |
| `Circuit breaker active` | 連続損失で保護発動 | クールダウン期間を待つかリセット |
